# 🐾  暴力喵喵字典提取器 — 日文专有名词自动提取工具

一款针对日文视觉小说 / 轻小说文本，通过**多轮暴力滑窗切片 → 多阶段语法规则提纯 → AI 交叉校验**的方式，全自动提取人名、昵称、专有术语等专有名词并生成日译中字典的工具链。

## 程序步骤概览 

本程序分 **四个步骤**，依次执行：

<aside>
💡

初次使用时建议先放入一个小型 JSON 文件，每个阶段结束后查看工作目录中新生成了哪些文件，熟悉整体流程。

</aside>

### 步骤 1 · 暴力切片与词频分析

对输入 JSON 中的 `name` 和 `message` 字段，执行滑窗暴力切片（可配置 `MIN_LEN` ~ `MAX_LEN`），统计所有子串词频，再经过 **六重过滤** 逐步提纯：

| 过滤阶段 | 说明 |
| --- | --- |
| ① 基础频率过滤 | 去除低于 `MIN_COUNT` 的低频噪声 |
| ② 去冗余过滤 | 频次差 < `TOLERANCE` 的子串视为父串冗余，自动剔除 |
| ③ 前后缀过滤 | 识别因助词（は/が/の/に 等）粘连产生的虚假高频词 |
| ④ 黑名单过滤 | 精确匹配 `黑名单.txt` 中的已知噪声词 |
| ⑤ 连续重复过滤 | 剔除如 `ああああ` 等重复拟声/感叹碎片 |
| ⑥ 连坐黑名单过滤 | 包含任意连坐关键字的词条一律移除 |

完成后自动进入**字典后期清理**，结合 `项目GPT字典.txt` 的姓名组合和修饰规则进行深度去杂。

**输出：** `初版词频/词频.txt` + `待人工查看/` 可疑记录

### 步骤 2 · AI 词频清洗

将第一阶段的词频列表分块发送给 AI 模型，由模型识别并标记**纯语法碎片、噪音词、通用日常名词**（即不可能是专有名词的部分），再进行交叉比对过滤。

- 支持**多次提取**（默认 2 次），取交集来降低误删概率
- 支持**双模型配置**，奇偶次分别使用不同的 API 节点
- 逐块校验：AI 返回的每个词条 ID 和原词必须严格匹配，不一致即自动重试
- 自动断点续传：已完成的分块缓存在 `缓存/过滤缓存/` 下，重跑时自动跳过

**输出：** `已过滤成功词频/词频.txt`（提纯后的词频表）

### 步骤 3 · 语境区块提取

根据提纯后的词频表，从原始语料中智能截取含目标词的**上下文区块**，作为第四阶段 AI 提取的输入素材。

- **三阶段递进提取**：严格 → 宽松 → 兜底，确保每个目标词都有足够语境覆盖
- **智能扩展**：当区块边缘命中其他目标词时，自动向前/后拓展上下文
- **最优装箱合并**：多个小区块合并入同一个 CSV，减少 API 请求数
- **父串感知**：避免短词被长词的出现位置误匹配
- 支持双轮生成（不同行数配置），分别输出至 `生成字典用/1/` 和 `生成字典用/2/`

**输出：** `生成字典用/{轮次}/request_batch_XXXX.csv` + 对应 `.txt` 提示词

### 步骤 4 · AI 字典生成

将语境区块分块发送给 AI，提取**人名、昵称、称呼、专有术语**等并生成日译中字典。

- AI 返回 TSV 格式，经过**格式校验 + 来源校验**（每个"日文原词"必须存在于原始输入中）
- 支持 `.txt` 提示文件注入额外上下文（如高频词表）
- 全部提取完成后自动进入**后处理流程**：
    - 去重合并（支持男/女性别双版本保留）
    - 黑名单 / 人名黑名单 / 昵称黑名单过滤
    - 全文词频统计与排序
    - 未命中词频查漏比对

**输出：** `项目GPT字典-生成.txt`（最终字典）+ `未命中词频.txt`
**注：** 经测试，会有部分ai因模型能力问题总是输出不存在于原始输入的词组导致报错，单文件缓存中止，遇到此问题时更换模型，或自行去对应文件夹修改ai幻觉的词组
---


### 2. 准备输入数据

将文本转换为 JSON 数组格式，放入 `[工作目录]/输入json/`：只支持name，message键值

```json
[
  { "name": "太郎", "message": "今日はいい天気ですね。" },
  { "name": "花子", "message": "そうですね、ダンジョンに行きましょう。" }
  { "message": "そうですね、ダンジョンに行きましょう。" }
]
```

### 4. 按顺序运行

依次点击**步骤 1 → 步骤 2 → 步骤 3 → 步骤 4** 即可。最终字典输出在 `[工作目录]/项目GPT字典-生成.txt`。

## 关于模型选择

- 字典提取对模型智力要求不高，**DeepSeek 起步即可**，大部分模型都能提取完整
- 好模型的优势在于**翻译准确性更高**，省去人工核对原文的时间
- 可通过修改 `Prompts.py` 中的提示词来优化模型对通用名词的过滤效果
- 遇到不想要的词条可加入**生成字典黑名单**，逐步优化（水滴石穿）

<aside>
⚠️

**关于提取质量**：本项目优先保证**全覆盖**（不漏提），但无法完全保证**零噪声**（不杂）。

</aside>

## 关于 AI 调用次数

| 配置 | 请求次数估算 |
| --- | --- |
| 默认（2 轮 × 各 2 次） | 约 **300 次** |
| 精简模式（各 1 次） | 约 **150 次** |
- **步骤 2**（词频清洗）和**步骤 4**（字典生成）会调用 AI
- **强烈建议提取 2 次**，通过交集过滤来降低误删/误提概率

### 双模型配置

适用于想用好模型为主、白嫖模型兜底的场景：

- `modelName` — 填主力好模型
- `modelName1` — 填次要/免费模型（对应 `endpoint1` 和 `token1`）

脚本会优先采信主模型的结果。

## 注意事项

### 提供基础字典可显著增强效果

在工作目录下放置 `项目GPT字典.txt` 后，步骤 1 的**不分割词保护**和**字典后期清理**都会读取它。提供得越全面，去杂效果越好，API token 消耗也越少。

#### 为什么强烈推荐提供项目字典？

项目字典对**切片保护**和**后期去杂**两个环节都有关键作用，用中文类比最直观：

**1. 切片保护——防止专有名词被滑窗切碎**

假设文本中有人名「小明」，滑窗暴力切片会对所有连续子串计数。如果「小明」不在字典中，它就会被拆成 `小`、`明` 等无意义碎片混入统计。而一旦「小明」被加入项目字典，它就会被标记为**不分割词**——整个词在切片前被替换为单字符占位符，滑窗根本碰不到它的内部结构，确保「小明」始终以**完整形态**进入频次表。

**2. 后期去杂——自动排除人名粘连的语法噪声**

字典后期清理阶段会用项目字典中的词条自动构建排除规则。例如字典中有「小明」，系统会自动识别并排除：

- `小明啊`、`小明呢`、`小明的` — 人名 + 语气词/助词的粘连碎片
- `和小明`、`跟小明` — 虚词 + 人名的粘连碎片
- `小明小红`、`小红小明` — 姓名两两组合的拼接噪声

这些都是滑窗统计中极其常见的高频噪声。**字典越全，自动排除的噪声越多**，最终词频表越干净，后续 AI 提取的 token 消耗也越低。

<aside>
💡

简单来说：没有项目字典时，工具也能跑，但会多出大量「人名+助词」的噪声词条需要ai清理。提供字典后，这些噪声会被**自动精确剔除**，省时省力。

</aside>

### Galgame 项目获取基础字典

1. 搜索 **【游戏名】+ vndb**（可安装 vndb 中文脚本插件）
2. 进入游戏页面，点击下方**人物标签**，复制所有人物词条
3. 配合 `喵喵小工具/vndb` 文件夹下的提示词发给 AI
4. AI 回复的内容即可作为基础字典

<aside>
⚠️

vndb 词条有极小概率拼写错误，请稍加留意。

</aside>

### 小说项目获取基础字典

1. 先运行**步骤 1**，查看词频最高的 name 字段词条
2. 将确认的人名/术语复制到 `项目GPT字典.txt` 中

<aside>
💡

可让 AI 写个小脚本，将你的每行文本转为 JSON 的 `message` 格式。若单行过长，可按句号切分。

</aside>

### 调试模式

在 `设置.yaml` 中设置 `DEBUG_MODE: true`，运行时可输入要追踪的词语，脚本会在每个过滤阶段打印该词的存活/被过滤状态及原因，方便排查问题。
