## 完整处理示例

假设 JSON 输入中有以下一条数据：

```json
{ "name": "エリス", "message": "「エリスはダンジョンの奥に行った！素晴らしい」" }
```

相关配置文件内容（示例）：

| 文件 | 内容（部分） |
| --- | --- |
| `项目GPT字典.txt` | `エリス`、`ダンジョン` |
| `首尾符号.txt` | `「`、`」`、`…`、`。` |
| `文本符号分割.txt` | `、`、`。`、`！`、`？` |
| `开头过滤.txt` | `っ`、`ッ`、`ー` |
| `结尾过滤.txt` | `の`、`を`、`は`、`が`、`に`、`へ`、`と`、`で` |

## 阶段一：`run_analysis()` — 词频分析

### 1. JSON 文本扫描阶段（滑窗统计）

| 文件 | 使用时机 | 具体作用 |
| --- | --- | --- |
| `统一不分割的词.txt` | 扫描每条 message 时**最先使用** | 用正则把不分割词替换为占位符，防止被滑窗切碎 |
| `项目GPT字典.txt` | 同上（词条被追加到不分割词列表） | 同上 |
| `白名单.txt` | 同上（词条也被追加到不分割词列表） | 同上 |
| `首尾符号.txt` | 替换占位符后、**分割前** | 从每段文本的首尾反复剥离这些符号 |
| `文本符号分割.txt` | 首尾剥离后 | 按这些符号把文本切成多个片段 |
| `开头过滤.txt` | **滑窗遍历子串时** | 跳过以这些字符开头的子串，不计入频次 |
| `结尾过滤.txt` | **滑窗遍历子串时** | 跳过以这些字符结尾的子串，不计入频次 |

<aside>
📌

`name` 字段的处理略有不同：不走占位符替换和首尾剥离，只用分割符（额外包含 `・`）切分后直接整体计数，并加入 name 保护集。

</aside>

#### message 字段处理示例

以 `「エリスはダンジョンの奥に行った！素晴らしい」` 为例，逐步演示：

| 步骤 | 处理前 | 加载的 txt | 命中字符 | 处理后 |
| --- | --- | --- | --- | --- |
| **① 不分割词替换** | `「エリスはダンジョンの奥に行った！素晴らしい」` | `项目GPT字典.txt` | `エリス` → ⓪
`ダンジョン` → ① | `「⓪は①の奥に行った！素晴らしい」`
（同时 `エリス` 和 `ダンジョン` 各 +1 频次） |
| **② 首尾符号剥离** | `「⓪は①の奥に行った！素晴らしい」` | `首尾符号.txt` | `「`（首）、`」`（尾） | `⓪は①の奥に行った！素晴らしい` |
| **③ 文本符号分割** | `⓪は①の奥に行った！素晴らしい` | `文本符号分割.txt` | `！` | 片段A: `⓪は①の奥に行った`
片段B: `素晴らしい` |
| **④ 滑窗切片** | 对每个片段生成所有 `MIN_LEN`~`MAX_LEN` 长度子串 | `开头过滤.txt`
`结尾过滤.txt` | 见下方详细表 | 还原占位符后计入频次 |

#### 滑窗阶段详细示例（片段A 的部分长度=2 子串）

| 子串（占位符态） | 还原后 | 检查项 | 结果 |
| --- | --- | --- | --- |
| `⓪は` | `エリスは` | 尾字 `は` → 命中 `结尾过滤.txt` | ❌ 跳过，不计入频次 |
| `は①` | `はダンジョン` | 首尾均未命中过滤规则 | ✅ `はダンジョン: +1` |
| `①の` | `ダンジョンの` | 尾字 `の` → 命中 `结尾过滤.txt` | ❌ 跳过，不计入频次 |
| `の奥` | `の奥` | 首尾均未命中过滤规则 | ✅ `の奥: +1` |
| `奥に` | `奥に` | 尾字 `に` → 命中 `结尾过滤.txt` | ❌ 跳过，不计入频次 |
| `に行` | `に行` | 首尾均未命中过滤规则 | ✅ `に行: +1` |
| `行っ` | `行っ` | 首尾均未命中过滤规则 | ✅ `行っ: +1` |
| `った` | `った` | 首尾均未命中过滤规则 | ✅ `った: +1` |

<aside>
📌

`エリス` 和 `ダンジョン` 在步骤①替换时已被**单独整体计数 +1**，不会再通过滑窗重复统计。这就是"不分割词保护"的核心机制——确保它们以完整形态进入频次表。

</aside>

### 2. 多阶段过滤（扫描完成后）

| 过滤阶段 | 使用的文件 | 使用时机 |
| --- | --- | --- |
| 阶段 1：基础频率过滤 | 无额外 txt（用 `设置.yaml` 中的 `MIN_COUNT`） | 滑窗统计结束后，**第一轮筛选** |
| 阶段 2：去冗余过滤 | 无额外 txt（用 `TOLERANCE` 参数） | 频率过滤之后 |
| 阶段 3：前后缀过滤 | `加前后词缀一处.txt` | 去冗余之后，检查首尾单字符是否为"虚词缀" |
| 阶段 4：黑名单过滤 | `黑名单.txt` | 前后缀过滤之后，精确匹配删除 |
| 阶段 5：连续重复过滤 | 无额外 txt（用 `REPEAT_THRESHOLD`） | 黑名单之后 |
| 阶段 6：连坐黑名单过滤 | `连坐黑名单.txt` | **最后一轮过滤**，包含这些字符的即连坐删除 |

<aside>
📌

`白名单.txt` 在阶段 3～6 中也被使用——白名单词条会**跳过**这些过滤，起保护作用。

</aside>

#### 阶段 2 举例：去冗余过滤

假设滑窗统计后得到如下词频：

| 词条 | 频次 | 关系 |
| --- | --- | --- |
| `艾莉丝` | 50 | 父串（先被选中） |
| `莉丝` | 53 | `艾莉丝` 的子串 |

**判定逻辑**：`莉丝` 是 `艾莉丝` 的子串，频次差 = 53 − 50 = **3** < `TOLERANCE`(7)

→ 说明 `莉丝` 几乎只在 `艾莉丝` 内部出现，属于冗余碎片 → ❌ **删除 `莉丝`**

**反例**：如果 `莉丝` 频次为 60，差值 = 60 − 50 = 10 ≥ 7 → 说明它还独立出现了很多次 → ✅ **保留**

<aside>
🛡️

保护机制：如果 `莉丝` 曾出现在 JSON 的 `name` 字段中（属于 name 保护集），即使频次差 < 7 也**不会被去冗余删除**。

</aside>

#### 阶段 3 举例：前后缀过滤

假设 `加前后词缀一处.txt` 中包含 `に`、`が`、`は`：

| 词条 | 频次 | 去掉前/后缀后 | 去缀后频次 | 差值 | 结果 |
| --- | --- | --- | --- | --- | --- |
| `にエリス` | 30 | 前缀 `に` → `エリス` | 50 | 20 ≥ `MIN_DIFF`(20) | ❌ 删除 |
| `エリスが` | 35 | 后缀 `が` → `エリス` | 50 | 15 < `MIN_DIFF`(20) | ✅ 保留 |

**逻辑**：类似去冗余，但只检查首尾**单个字符**是否属于指定虚词缀集合，且频次差阈值默认从 7 提高到 **20**（`MIN_PREFIX_SUFFIX_COUNT_DIFF`）。

#### 阶段 4 举例：黑名单过滤

`黑名单.txt` 内有 `気持ち`、`学園`：

| 词条 | 是否在黑名单中 | 是否在白名单中 | 结果 |
| --- | --- | --- | --- |
| `気持ち` | ✅ 是 | 否 | ❌ 删除 |
| `学園` | ✅ 是 | ✅ 是（用户手动保护） | ✅ 保留（白名单优先） |

#### 阶段 5 举例：连续重复过滤

`REPEAT_THRESHOLD` = 4，即同一片段连续重复 4 次及以上即删除：

| 词条 | 重复模式 | 结果 |
| --- | --- | --- |
| `ああああ` | `あ` × 4 次 | ❌ 删除 |
| `はぁはぁはぁはぁ` | `はぁ` × 4 次 | ❌ 删除 |
| `ふふふ` | `ふ` × 3 次（< 4） | ✅ 保留 |

#### 阶段 6 举例：连坐黑名单过滤

`连坐黑名单.txt` 内有 `ちゃん`、`くん`：

| 词条 | 包含的连坐关键字 | 是否在白名单中 | 结果 |
| --- | --- | --- | --- |
| `お姉ちゃん` | 含 `ちゃん` | 否 | ❌ 连坐删除 |
| `カズくん` | 含 `くん` | 否 | ❌ 连坐删除 |
| `エリちゃん` | 含 `ちゃん` | ✅ 是（用户手动保护） | ✅ 保留（白名单优先） |

### 3. 误过滤分析 & 写入结果

过滤完成后，脚本会自动比对 `name` 保护集中的词条与最终存活词条——凡是出现在 name 中但被过滤掉的词条，会被记录到 `输出的name段可能误过滤.txt`，方便人工复查。

**输出文件：**

- `初版词频/词频.txt` — 最终词频表
- `待人工查看/输出的name段可能误过滤.txt` — 可疑误过滤记录
- `待人工查看/已删除.txt` — 阶段二的详细删除日志

阶段一结束。

## 阶段二：`run_post_processing()` — 字典后期清理

| 文件 | 使用时机 | 具体作用 |
| --- | --- | --- |
| `项目GPT字典.txt` | 阶段二开始时重新读取 | 提取第一列作为 Base 词库 + 生成姓名两两组合排除集 |
| `字典后期处理.txt`（`./txt/` 下） | 紧接着读取 | 单列行 → 后缀扩展规则；双列行（前/后/前后/包含）→ 修饰过滤规则 |

### 词库构建流程

1. 从 `项目GPT字典.txt` 提取第一列 → **Base 词库**（如 `エリス`、`ダンジョン`）
2. Base 词库两两组合 → **姓名排除集**（如 `エリスダンジョン`、`ダンジョンエリス`）
3. 从 `字典后期处理.txt` 读取**后缀扩展规则**（单列行，如 `さん`、`くん`、`様`）
4. Base 词库 × 后缀 → **扩展词库**（如 `エリスさん`、`ダンジョンさん`）
5. Base 词库 + 扩展词库 = **全量判定词库**

### 深度过滤规则

对阶段一输出的 `词频.txt` 逐行扫描，按以下优先级过滤：

| 规则 | 检查方式 | 示例 |
| --- | --- | --- |
| **姓名组合匹配** | 词条 ∈ 姓名排除集 → 完全匹配删除 | `日向琴子` ∈ 排除集 → ❌ 删除 |
| **の字过滤** | 词条以 `の` 开头或结尾 → 直接删除 | `エリスの` → ❌ 删除 |
| **完全匹配** | 词条 ∈ 全量判定词库 → 删除 | `エリスさん` ∈ 扩展词库 → ❌ 删除 |
| **修饰规则过滤** | 词条中含有修饰字符，且拆分后的另一部分 ∈ 全量判定词库 | 规则 `と` `前`：词条 `とエリス` 含 `と`  • `エリス` ∈ 词库 → ❌ 删除 |

#### 修饰规则举例

`字典后期处理.txt` 中的双列行定义了四种匹配模式：

| 规则内容 | 模式 | 含义 | 示例命中 |
| --- | --- | --- | --- |
| `と` `前` | 前匹配 | 找到 `と` 后，检查其**右侧**是否含有词库词条 | `とエリス` → `と` 右侧有 `エリス` → ❌ |
| `と` `后` | 后匹配 | 找到 `と` 后，检查其**左侧**是否含有词库词条 | `エリスと` → `と` 左侧有 `エリス` → ❌ |
| `と` `前后` | 双向匹配 | 同时检查 `と` 的左侧和右侧 | 以上两种情况都命中 → ❌ |
| `の` `包含` | 包含匹配 | 只要词条含 `の` 且同时含任意词库词条 | `エリスの奥` 含 `の` 且含 `エリス` → ❌ |

最终过滤结果回写至 `初版词频/词频.txt`，被删除的词条详细日志写入 `待人工查看/已删除.txt`（按**规则过滤、の字过滤、完全匹配**三类分组，各类内按频率降序排列）。

## 全流程一览图


JSON 输入
  │
  ├─ name 字段 ─→ 分割符切分 ─→ 整体计数 + 加入 name 保护集
  │
  └─ message 字段
       │
       ├─ ① 不分割词替换（项目GPT字典 + 统一不分割的词 + 白名单）
       │    └─ 匹配到的词整体计数 +1，替换为单字符占位符
       │
       ├─ ② 首尾符号剥离（首尾符号.txt）
       │
       ├─ ③ 文本符号分割（文本符号分割.txt）
       │
       └─ ④ 滑窗切片（MIN_LEN ~ MAX_LEN）
            ├─ 跳过：以 开头过滤字符 开头的子串
            ├─ 跳过：以 结尾过滤字符 结尾的子串
            └─ 还原占位符 → 累加频次
                 │
                 ▼
         ┌─ 阶段1：基础频率过滤（MIN_COUNT）
         ├─ 阶段2：去冗余过滤（TOLERANCE）    ← name 保护集豁免
         ├─ 阶段3：前后缀过滤（MIN_DIFF）      ← 白名单豁免
         ├─ 阶段4：黑名单精确匹配              ← 白名单豁免
         ├─ 阶段5：连续重复过滤                ← 白名单豁免
         ├─ 阶段6：连坐黑名单过滤              ← 白名单豁免
         │
         ├─ 输出：初版词频/词频.txt
         ├─ 输出：name段可能误过滤.txt
         │
         ▼
  阶段二：字典后期清理
         ├─ 读取 项目GPT字典.txt → Base 词库 + 姓名组合
         ├─ 读取 字典后期处理.txt → 后缀规则 + 修饰规则
         ├─ 构建全量判定词库
         ├─ の字过滤 / 完全匹配 / 修饰规则过滤
         │
         ├─ 回写：词频.txt（精简版）
         └─ 输出：已删除.txt（详细日志）
